## Detaillierte Erklärungen

### 1. Kontoerstellung
Ein GitHub-Konto ist Ihr persönlicher Zugang zur Plattform. Es ermöglicht Ihnen, Repositories zu erstellen, zu bearbeiten und an Projekten mitzuarbeiten. Mit einem Konto können Sie auch Issues erstellen, Pull Requests einreichen und mit anderen Entwicklern interagieren. Die Zwei-Faktor-Authentifizierung (2FA) fügt eine zusätzliche Sicherheitsebene hinzu, indem sie neben dem Passwort einen zweiten Faktor zur Anmeldung erfordert.

**Übung**: 
1. Gehen Sie zu [github.com](https://github.com) und klicken Sie auf "Sign up".
2. Folgen Sie den Anweisungen zur Kontoerstellung.
3. Richten Sie die Zwei-Faktor-Authentifizierung (2FA) ein.

### 2. Repositories
Ein Repository ist wie ein digitaler Ordner für Ihr Projekt. Es enthält alle Projektdateien sowie die gesamte Versionshistorie. Repositories ermöglichen es Ihnen, Änderungen zu verfolgen, mit anderen zusammenzuarbeiten und Ihr Projekt zu organisieren. Sie können öffentliche Repositories für Open-Source-Projekte oder private für sensible Arbeit erstellen.

**Übung**:
1. Besuchen Sie das [NADOO-Wiki Repository](https://github.com/NADOOIT/NADOO-Wiki).
2. Schauen Sie sich die Struktur und den Inhalt des Repositories an. Beachten Sie die verschiedenen Dateien und Ordner.

### 3. Forking und Branching
**Forking** erstellt eine vollständige Kopie eines Repositories unter Ihrem eigenen GitHub-Konto. Dies ist besonders nützlich, wenn Sie an einem Projekt mitarbeiten möchten, für das Sie keine direkten Schreibrechte haben. Forking ermöglicht es Ihnen, frei Änderungen vorzunehmen, ohne das Original-Repository zu beeinflussen. Es ist ein wesentlicher Bestandteil des Open-Source-Beitragsmodells.

**Branching** erstellt eine separate Entwicklungslinie innerhalb eines Repositories. Es ermöglicht Ihnen, an verschiedenen Funktionen oder Experimenten zu arbeiten, ohne den Hauptcode zu beeinflussen. Branches sind besonders nützlich für die Entwicklung neuer Funktionen, das Beheben von Fehlern oder das Testen von Ideen.

**Übung**:
1. Klicken Sie auf den "Fork" Button oben rechts auf der NADOO-Wiki Repository-Seite.
2. In Ihrem geforkten Repository, klicken Sie auf "Branch: main" und geben Sie "meine-verbesserungen" ein, um einen neuen Branch zu erstellen.

### 4. GitHub Issues
Issues sind ein Werkzeug zur Verfolgung von Aufgaben, Verbesserungen und Fehlern in Ihrem Projekt. Sie dienen als eine Art To-Do-Liste und Diskussionsplattform. Mit Issues können Sie Aufgaben zuweisen, Labels zur Kategorisierung verwenden und Fortschritte verfolgen. Sie fördern auch die Zusammenarbeit, indem sie einen zentralen Ort für Diskussionen über spezifische Aspekte des Projekts bieten.

**Übung**:
1. Gehen Sie zum "Issues" Tab im NADOO-Wiki Repository.
2. Erstellen Sie ein neues Issue mit dem Titel "Vorschlag zur Verbesserung der Dokumentation".
3. Beschreiben Sie kurz eine Idee zur Verbesserung und klicken Sie auf "Submit new issue".

### 5. Pull Requests
Pull Requests sind ein Mechanismus, um Änderungen aus einem Branch (oder Fork) in einen anderen Branch zu integrieren. Sie ermöglichen es Entwicklern, ihre Änderungen vorzuschlagen und um eine Überprüfung zu bitten, bevor sie in den Hauptcode integriert werden. Pull Requests bieten eine Plattform für Code-Reviews, Diskussionen und weitere Verfeinerungen, bevor Änderungen akzeptiert werden.

**Übung**:
1. In Ihrem geforkten Repository, bearbeiten Sie eine Datei (z.B. README.md).
2. Erstellen Sie einen Pull Request, indem Sie auf "Contribute" und dann "Open pull request" klicken.
3. Überprüfen Sie Ihre Änderungen und klicken Sie auf "Create pull request".

### 6. Sterne vergeben und Repositories beobachten
**GitHub-Sterne** sind eine einfache Möglichkeit, interessante Repositories zu markieren und deren Popularität zu messen. Es wird empfohlen, alle Repositories, die Sie im Rahmen dieses Bildungsgangs bearbeiten, mit einem Stern zu versehen.

**Repositories beobachten** ermöglicht es Ihnen, Benachrichtigungen über neue Aktivitäten zu erhalten. Dies ist nützlich, um über Änderungen, neue Issues oder Pull Requests auf dem Laufenden zu bleiben, ohne aktiv am Projekt mitzuarbeiten.

**Übung**:
1. Auf der NADOO-Wiki Repository-Seite, klicken Sie auf den "Star" Button oben rechts.
2. Klicken Sie auf "Watch" und wählen Sie, wie Sie benachrichtigt werden möchten.

## Vorschläge für Issues

Im Rahmen dieses Bildungsgangs sollen Sie auch lernen, Issues zu erstellen. Hier einige Vorschläge:

- Lokalisierung: Übersetzen Sie Teile der Dokumentation in eine andere Sprache.
- Videotutorial: Schlagen Sie die Erstellung eines kurzen Video-Tutorials vor.
- Verbesserte Dokumentation: Identifizieren Sie Bereiche, die mehr Erklärung benötigen.
- Code-Optimierung: Schlagen Sie Verbesserungen für bestehenden Code vor.
- Automatisierte Tests: Diskutieren Sie die Möglichkeit, Tests hinzuzufügen.
