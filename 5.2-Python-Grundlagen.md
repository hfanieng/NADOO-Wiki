## 1. Python-Installation und virtuelle Umgebung

Bevor wir mit dem Coding beginnen, installieren wir Python 3.11 mithilfe von uv und erstellen eine virtuelle Umgebung.

### Python 3.11 Installation mit uv

1. Öffnen Sie das Terminal in Visual Studio Code (View > Terminal oder ``Ctrl+` ``).

2. Installieren Sie uv:

   Für macOS und Linux:
   ```bash
   curl -LsSf https://astral.sh/uv/install.sh | sh
   ```

   Für Windows (in PowerShell):
   ```powershell
   irm https://astral.sh/uv/install.ps1 | iex
   ```

3. Installieren Sie Python 3.11:
   ```bash
   uv python install 3.11
   ```

4. Überprüfen Sie die Installation:
   ```bash
   uv python --version
   ```

### Erstellen einer virtuellen Umgebung

1. Erstellen Sie eine virtuelle Umgebung:
   ```bash
   uv venv -p 3.11
   ```

2. Aktivieren Sie die virtuelle Umgebung:

   Für macOS und Linux:
   ```bash
   source .venv/bin/activate
   ```

   Für Windows:
   ```bash
   .\.venv\Scripts\activate


## 2. Python Grundkonzepte

Hier ist ein umfassendes Beispielskript, das grundlegende Python-Konzepte erklärt:

```python
# Dies ist ein umfassendes Beispielskript, das grundlegende Python-Konzepte erklärt.
# Zeilen, die mit '#' beginnen, sind Kommentare und werden von Python nicht ausgeführt.

# 1. Variablen und Zuweisungen
# Variablen sind wie Behälter, in denen wir Daten speichern können.
zahl = 10  # Integer (Ganzzahl)
kommazahl = 3.14  # Float (Gleitkommazahl)
text = "Hallo, Welt!"  # String (Zeichenkette)
liste = [1, 2, 3, 4, 5]  # Liste (veränderbare Sequenz von Elementen)

# Bei Zuweisungen wird zuerst der Ausdruck rechts vom '=' ausgewertet, 
# dann wird das Ergebnis der Variable links zugewiesen.
a = b = c = 1 + 2 * 3  # Zuerst 2*3, dann 1+6, dann Zuweisungen von rechts nach links
print(f"a = {a}, b = {b}, c = {c}")

# 2. Funktionen
# Funktionen sind wiederverwendbare Codeblöcke.
def addiere(a, b):
    return a + b  # 'return' gibt das Ergebnis der Funktion zurück

# Funktionen werden erst bei Aufruf ausgeführt, nicht bei der Definition
ergebnis = addiere(5, 3)
print(f"5 + 3 = {ergebnis}")

# 3. Kontrollstrukturen
# If-Anweisung: Bedingte Ausführung von Code
if zahl == 10:  # '==' wird für Vergleiche verwendet, '=' für Zuweisungen
    print("Die Zahl ist 10")
else:
    print("Die Zahl ist nicht 10")

# For-Schleife: Iteration über eine Sequenz
for i in range(5):  # 'range(5)' erzeugt die Zahlen 0, 1, 2, 3, 4
    print(f"Schleifendurchlauf: {i}")

# While-Schleife: Wiederholung, solange eine Bedingung wahr ist
counter = 0
while counter < 3:
    print(f"While-Schleife: Zähler = {counter}")
    counter += 1  # Erhöht den Zähler um 1

# 4. Klassen und Objekte
# Eine Klasse ist wie ein Bauplan für Objekte.
class Auto:
    def __init__(self, marke, modell):
        # '__init__' ist der Konstruktor, aufgerufen bei Objekterstellung
        # 'self' bezieht sich auf das aktuelle Objekt
        self.marke = marke
        self.modell = modell
    
    def fahren(self):
        print(f"{self.marke} {self.modell} fährt.")

# Objekterstellung und -verwendung
mein_auto = Auto("VW", "Golf")
mein_auto.fahren()

# 5. Gültigkeitsbereich (Scope)
globale_variable = "Ich bin global"

def funktion_mit_lokaler_variable():
    lokale_variable = "Ich bin lokal"
    print(globale_variable)  # Zugriff auf globale Variable ist möglich
    print(lokale_variable)

funktion_mit_lokaler_variable()
# print(lokale_variable)  # Dies würde einen NameError verursachen

# 6. Ausführungsreihenfolge und Besonderheiten
print("\nAusführungsreihenfolge und Besonderheiten:")

def beispiel_funktion():
    print("Diese Funktion wurde aufgerufen")

# Funktionsdefinitionen werden gelesen, aber nicht sofort ausgeführt
print("1. Dies wird zuerst ausgeführt")

if False:
    print("2. Dieser Code wird nie ausgeführt")

for i in range(0):
    print("3. Dieser Code wird auch nie ausgeführt")

print("4. Dies wird als Zweites ausgeführt")

# Jetzt rufen wir die Funktion auf
beispiel_funktion()

print("\nProgramm beendet")
```

## 3. Erklärung der Konzepte

### 3.1 Variablen und Zuweisungen

Variablen in Python sind wie Behälter, die Daten speichern. Sie müssen nicht explizit deklariert werden und können verschiedene Datentypen enthalten:

- `Integer`: Ganze Zahlen (z.B. `zahl = 10`)
- `Float`: Gleitkommazahlen (z.B. `kommazahl = 3.14`)
- `String`: Zeichenketten (z.B. `text = "Hallo, Welt!"`)
- `Liste`: Geordnete Sammlungen von Elementen (z.B. `liste = [1, 2, 3, 4, 5]`)

Bei Zuweisungen wird zuerst der Ausdruck rechts vom `=` ausgewertet, dann das Ergebnis der Variable links zugewiesen. Mehrfachzuweisungen sind möglich:

```python
a = b = c = 1 + 2 * 3
```

Hier wird zuerst `2 * 3` berechnet, dann `1 + 6`, und schließlich wird 7 allen drei Variablen zugewiesen.

### 3.2 Funktionen

Funktionen sind wiederverwendbare Codeblöcke. Sie werden mit `def` definiert:

```python
def addiere(a, b):
    return a + b
```

Funktionen werden erst bei Aufruf ausgeführt, nicht bei der Definition. Der `return`-Befehl gibt das Ergebnis der Funktion zurück.

### 3.3 Kontrollstrukturen

#### If-Anweisung
Bedingte Ausführung von Code:

```python
if bedingung:
    # Code, der ausgeführt wird, wenn die Bedingung wahr ist
else:
    # Code, der ausgeführt wird, wenn die Bedingung falsch ist
```

#### For-Schleife
Iteration über eine Sequenz:

```python
for element in sequenz:
    # Code, der für jedes Element ausgeführt wird
```

#### While-Schleife
Wiederholung, solange eine Bedingung wahr ist:

```python
while bedingung:
    # Code, der wiederholt wird, solange die Bedingung wahr ist
```

### 3.4 Klassen und Objekte

Klassen sind Blaupausen für Objekte. Sie können Attribute (Daten) und Methoden (Funktionen) enthalten:

```python
class Auto:
    def __init__(self, marke, modell):
        self.marke = marke
        self.modell = modell
    
    def fahren(self):
        print(f"{self.marke} {self.modell} fährt.")
```

`self` bezieht sich auf die aktuelle Instanz der Klasse und ermöglicht den Zugriff auf Attribute und Methoden des Objekts.

### 3.5 Gültigkeitsbereich (Scope)

Variablen haben unterschiedliche Gültigkeitsbereiche:
- Globale Variablen sind im gesamten Programm sichtbar.
- Lokale Variablen sind nur innerhalb der Funktion sichtbar, in der sie definiert wurden.

### 3.6 Ausführungsreihenfolge

1. Python-Code wird von oben nach unten gelesen und interpretiert.
2. Die tatsächliche Ausführung folgt nicht immer dieser linearen Reihenfolge:
   - Funktionen und Klassen werden definiert, aber erst bei Aufruf ausgeführt.
   - Bedingte Anweisungen und Schleifen können Teile des Codes überspringen.
3. Bei Zuweisungen und Ausdrücken erfolgt die Auswertung von rechts nach links.
4. In komplexeren Ausdrücken gelten die üblichen Regeln der Operatorpriorität.

Diese Grundlagen bilden das Fundament für das Verständnis und die Entwicklung von Python-Programmen. Mit diesen Konzepten können Sie beginnen, eigene Programme zu schreiben und komplexere Aufgaben zu lösen.
