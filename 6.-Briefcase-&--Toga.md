### Wichtig: Beenden von laufenden Prozessen

Während der Arbeit mit den Tutorials werden Sie möglicherweise Situationen begegnen, in denen Sie einen laufenden Prozess stoppen müssen. Dies kann beispielsweise der Fall sein, wenn eine Anwendung in einer Endlosschleife hängt oder Sie den Entwicklungsserver beenden möchten. 

Um einen laufenden Prozess im Terminal zu stoppen, verwenden Sie die Tastenkombination:

**STRG + C** (oder **Control + C** auf Mac)

Diese Tastenkombination sendet ein Unterbrechungssignal an den aktuell laufenden Prozess und beendet ihn in der Regel sofort. Es ist eine wichtige Fähigkeit, die Sie häufig benötigen werden, besonders wenn Sie mit Entwicklungsservern oder lang laufenden Skripten arbeiten.


## Python 3.11 Installation mit uv

1. Öffnen Sie das Terminal in Visual Studio Code (View > Terminal oder ``Ctrl+` ``).

2. Installieren Sie uv mit folgendem Befehl:

   Für macOS und Linux:
   ```
   curl -LsSf https://astral.sh/uv/install.sh | sh
   ```

   Für Windows (in PowerShell):
   ```powershell
   irm https://astral.sh/uv/install.ps1 | iex
   ```

3. Installieren Sie Python 3.11 mit uv:
   ```
   uv python install 3.11
   ```

4. Überprüfen Sie die Installation:
   ```
   uv python --version
   ```
   Sie sollten eine Ausgabe wie "Python 3.11.x" sehen.

## Erstellen einer virtuellen Umgebung

### Was ist eine virtuelle Umgebung?

Eine virtuelle Umgebung ist ein isolierter Bereich auf Ihrem Computer, in dem Sie Python-Pakete und Abhängigkeiten für ein bestimmtes Projekt installieren können, ohne dass dies Auswirkungen auf andere Projekte oder das System-Python hat.

### Wozu sind virtuelle Umgebungen nützlich?

Virtuelle Umgebungen lösen mehrere Herausforderungen:

1. **Isolierung von Projekten**: Jedes Projekt kann seine eigenen Abhängigkeiten haben, ohne mit anderen Projekten in Konflikt zu geraten.
2. **Versionskontrolle**: Sie können verschiedene Versionen derselben Bibliothek für unterschiedliche Projekte verwenden.
3. **Einfache Reproduzierbarkeit**: Die Umgebung eines Projekts kann leicht auf anderen Systemen reproduziert werden.
4. **Saubere Entwicklungsumgebung**: Verhindert die Verschmutzung des globalen Python-Installationsverzeichnisses.
5. **Einfaches Teilen**: Erleichtert das Teilen von Projekten mit anderen Entwicklern, da alle Abhängigkeiten in einer Datei (z.B. requirements.txt) aufgelistet werden können.

Durch die Verwendung virtueller Umgebungen können Sie sicherstellen, dass Ihre Projekte konsistent und portabel bleiben, unabhängig davon, auf welchem System sie ausgeführt werden.

## Installation der virtuellen Umgebung

1. Erstellen Sie eine virtuelle Umgebung mit Python 3.11:
   ```
   uv venv -p 3.11
   ```

2. Aktivieren Sie die virtuelle Umgebung:

   Für macOS und Linux:
   ```
   source .venv/bin/activate
   ```

   Für Windows:
   ```
   .\.venv\Scripts\activate
   ```

3. Ihr Terminal-Prompt sollte sich ändern und (.venv) am Anfang anzeigen, was bedeutet, dass die virtuelle Umgebung aktiv ist.

### Was sind Briefcase und Toga?

**Briefcase** ist ein Werkzeug zur Erstellung plattformübergreifender Anwendungen in Python, das es Ihnen ermöglicht, Ihre Python-Projekte als native Anwendungen für verschiedene Betriebssysteme zu verpacken und zu verteilen. **Toga** ist ein plattformübergreifendes GUI-Toolkit für Python, das es Entwicklern ermöglicht, ansprechende Benutzeroberflächen zu erstellen, die auf verschiedenen Plattformen konsistent aussehen und sich gleich verhalten.
